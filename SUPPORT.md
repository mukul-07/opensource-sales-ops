# Getting Help

opensource-sales-ops is a community-maintained open source project. Here's how to get help efficiently.

## Where to ask

| Question type | Where |
|---|---|
| **Bug** (something is broken) | [GitHub Issues](https://github.com/mukul-07/opensource-sales-ops/issues) — use the Bug Report template |
| **Feature idea** | [GitHub Issues](https://github.com/mukul-07/opensource-sales-ops/issues) — use the Feature Request template |
| **How do I...?** | [GitHub Discussions](https://github.com/mukul-07/opensource-sales-ops/discussions) |
| **Setup help** | Check [docs/SETUP.md](docs/SETUP.md) and [USAGE.md](USAGE.md) first, then open a Discussion |
| **Security vulnerability** | [Report privately via GitHub Security Advisories](https://github.com/mukul-07/opensource-sales-ops/security/advisories/new) — see [SECURITY.md](SECURITY.md) |

## Before opening an issue

1. Search existing issues — someone may have reported it already
2. Run `npm run doctor` — it catches most setup problems
3. Include your OS, Node.js version, and the CLI you're using (Claude Code, OpenCode, etc.)

## What NOT to use GitHub Issues for

- General questions about outbound sales or SDR strategy
- Requests for personal sales coaching
- Support for modified forks or unofficial distributions

These will be closed and redirected to Discussions.
